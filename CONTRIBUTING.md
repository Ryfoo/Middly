# Contributing Guide

Thank you for your interest in contributing! All forms of contribution are welcome — no barriers, no gatekeeping.

---

## Ways to Contribute

- **Report a bug** — open an issue describing what happened and what you expected
- **Suggest a feature** — open an issue with your idea and why it would be useful
- **Fix a bug** — pick any open issue and submit a pull request
- **Improve documentation** — typos, clarity, missing examples — all welcome
- **Write tests** — more coverage is always appreciated
- **Refactor code** — clean up, simplify, or optimize existing code

---

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
   ```bash
   git clone https://github.com/Ryfoo/Middly.git
   cd Middly
   ```
3. **Create a branch** for your change
   ```bash
   git checkout -b your-branch-name
   ```
4. **Make your changes**
5. **Commit** with a clear message
   ```bash
   git commit -m "Short description of what you changed"
   ```
6. **Push** to your fork
   ```bash
   git push origin your-branch-name
   ```
7. **Open a Pull Request** against the `main` branch

---

## Guidelines

- Keep pull requests focused — one change per PR is easier to review
- Write clear commit messages

---

## No Contribution is Too Small

Fixing a typo, improving a comment, or adding a missing example — everything helps.

---

## Questions?

Open an issue and ask.
